# わんワード

犬用語のカタカナ4文字を当てる、Wordle風の1日1問パズルゲームです。HTML/CSS/JavaScriptのみで構築された静的サイトで、GitHub Pagesでの公開を想定しています。

## 概要

- 毎日1問、犬に関するカタカナ4文字の単語を出題するWordleクローン
- サーバーサイド処理・ビルドツール不要、ブラウザだけで完結する静的サイト
- GitHub Pagesにそのままホスティング可能

## 主な機能（想定）

- 1日1問の出題（日付をシードにして単語を自動選出）
- 5〜6トライ、文字ごとの正誤判定（完全一致・含む・不一致の色分け）
- カタカナ入力対応のオンスクリーンキーボードまたはテキスト入力
- 結果のシェア機能（絵文字グリッドなど）
- `localStorage` を使った正答履歴・連続正解数の保存

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（フレームワークなし）
- GitHub Pages（静的ホスティング）

## ディレクトリ構成（想定）

```
.
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── main.js         # ゲームのUI制御・判定ロジック
│   └── words.js        # 犬用語の単語リスト・日替わり選出ロジック
└── README.md
```

## セットアップ

ビルド不要の静的サイトです。リポジトリをクローンし、簡易HTTPサーバーで配信して確認します（`file://` で直接開くとブラウザによっては動作しない機能があるため推奨）。

```bash
# 例: Python の簡易サーバーを使う場合
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開いて確認してください。

## 使い方

ページを開くと、その日の犬用語（カタカナ4文字）が出題されます。制限回数以内に単語を入力して回答し、文字ごとの正誤ヒントを手がかりに正解を目指します。

## デプロイ（GitHub Pages）

1. GitHubにリポジトリを作成しpush
2. リポジトリの Settings > Pages で公開元ブランチ（例: `main` の `/root`）を指定する

## 開発

静的サイトのため自動テストは最小限です。単語選出ロジックなどを切り出した場合は、`tests/` に検証用のスクリプトを追加してください。
