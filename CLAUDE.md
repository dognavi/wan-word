# わんワード - CLAUDE.md

## プロジェクトの目的

犬用語のカタカナ4文字を当てる、1日1問のパズルゲーム「わんワード」。HTML/CSS/JSのみで動く静的サイトとして実装し、GitHub Pagesで公開する。実装計画は `.claude/plans/wanword-daily-puzzle.md`（ver.4）を参照。Phaseごとに実装を進め、各Phase完了時にユーザーの承認を得てから次に進む。

## 技術スタック

- HTML5 / CSS3 / Vanilla JavaScript（ビルドツール・フレームワーク不使用、外部ライブラリ・外部API不使用）
- テスト: Node標準の `node --test`（追加ライブラリなし）
- GitHub Pages（ホスティング）

## ディレクトリ構成の方針

- `index.html`: エントリーポイント
- `css/style.css`: スタイル
- `js/words.js`: 単語リスト（平文の配列）
- `js/day.js`: JSTの絶対日数を返す純粋関数（唯一の日付ソース）+ 出題インデックスの選出
- `js/judge.js`: 判定ロジック（2パスアルゴリズム）
- `js/keyboard.js`: 濁点/半濁点サイクルの純粋ロジック + キーボードのDOM生成
- `js/board.js`: 盤面描画・入力状態管理
- `js/candidates.js`: 候補46語の一覧表示UI
- `js/storage.js`: localStorage（当日の結果・streak）の読み書き
- `js/sanitize.js`: 安全なDOM生成ヘルパー
- `img/`: 背景画像などの画像アセット
- `test/*.test.js`: `node --test` で実行するテスト一式

## ゲームルール（確定仕様。逸脱不可）

- 文字種: カタカナのみ。「バ」「パ」は「ハ」と別の1文字。小さい文字（ャッェ等）・長音符「ー」も独立した1文字としてカウントする
- 単語長: 4文字固定 / 挑戦回数: 6回
- 判定ロジックの処理順序（**最重要**）:
  1. 全マスを走査し、位置も文字も一致するものを緑に確定。緑にした分だけ正解側の「その文字の在庫」を1減らす
  2. 残った入力文字を左から順に見て、在庫が残っていれば黄、在庫切れならグレー
- 辞書チェックはしない。カタカナ4文字が揃っていれば送信可能
- 出題は、JSTの絶対日数から固定シードの並べ替え索引表を経由して選ぶ。**単語リスト（`words.js`）の並び順とは無関係**であり、並び順を変えても出題結果に影響しない設計にする
- 候補46語は画面から確認できるようにする（絞り込みの推理を成立させるため、隠さない）
- 46語は46日で1巡する。一巡後は単語を追加していく運用とする
- `localStorage` に当日の結果と連続記録（streak）を保存する。streakは「現在の値」のみでベスト記録は保存しない

## コーディング規約

- 文字列比較の前には必ず `normalize('NFC')` を通す（濁点付き文字がNFC/NFDの2通りで表現されうるため）
- 文字分割は `Array.from()`（またはスプレッド構文）でコードポイント単位に行う。`str.length` は使わない
- JSTの絶対日数は `day.js` の一発計算式 `Math.floor((date.getTime() + 9*3600*1000) / 86400000)` のみで求める。Y/M/Dに分解してから組み立て直す実装はしない。Y/M/Dが必要なのは画面表示のみで、出題選出やstreak判定などのゲームロジックの意思決定には使わない
- 日付に関する計算（出題インデックス選出・streakの前日判定）は、必ず `day.js` の関数を経由する。`storage.js` 等で独自に日付計算をしない
- 負のmodは `normalizeMod(n, m) = ((n % m) + m) % m` を経由する（JSの `%` は数学的な剰余ではないため）
- 濁点/半濁点サイクルは行の言語規則に統一する（カ・サ・タ行=清音→濁音の2状態、ハ行=清音→濁音→半濁音の3状態）。実際にその文字が単語で使われているかどうかで例外を作らない
- 濁点キーが作用する対象は「カーソル位置（盤面上で現在最後に埋まっているマス）」と定義する
- タイル・キーボードの状態表現は色だけに依存せず、記号・テキストも併用する
- DOM描画は `sanitize.js` 経由の `textContent` / `createElement` のみで行う。`innerHTML` は使わない
- CSPにインラインstyle・インラインスクリプトを混在させない
- `localStorage` の読み書きは必ずtry/catchで包み、保存に失敗してもゲーム自体はプレイ可能な状態を維持する
- 「Wordle」という語は商標のため、コード内・表示・コミットメッセージ等で一切使用しない

## 単語リスト・出題順の限界（正直に明記する）

- 単語リストは難読化しない。候補として画面に公開するため、隠す意味がない
- 出題順は固定シードの並べ替え索引表による。これは「単語リストの並び順を見ただけで今日・明日の答えが分かってしまう」ことを防ぐものであり、**静的サイトである以上、ソースコードを実行すれば将来の答えも計算できてしまうことは防げない**。完全な秘匿ではないという前提を崩さない
